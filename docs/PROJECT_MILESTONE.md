# 招生智脑 B2B 平台 — 项目阶段性总结

> 更新日期: 2026-05-18 | 当前阶段: Phase 3 完成 | 下一阶段: 数据扩展

---

## 一、项目最终目标

构建一个面向中国高等院校的 **AI 招生 SaaS 平台**，帮助院校实现从"摆摊宣讲"到"24/7 AI 个性化招生咨询"的数字化转型。

### 终极能力矩阵

| 层面 | 能力 | 当前状态 |
|------|------|---------|
| **学生端** | 扫码 → AI 对话 → 画像生成 → 专业推荐 → 多校对比 | 🟢 核心闭环可用 |
| **院校端** | 品牌定制小程序 → 知识库管理 → 招生漏斗 → 学生画像看板 → 竞品分析 | 🟢 平台框架完整 |
| **数据层** | 10 省 × 800+ 院校录取/培养/就业数据 | 🔴 仅 3 省 162 校 |
| **商业层** | 付费订阅 + 模块开关 + 增值模块计费 | 🟢 技术框架就绪 |
| **部署层** | SaaS 多租户 + 旗舰版私有部署 | 🟡 SaaS 可用，私有部署未打包 |

## 四、数据扩展任务详解

### 4.1 现有项目流程需要什么数据

整个系统的数据链路有五层，每一层依赖不同的数据：

```
Layer 1: 院校基础信息 → College 表
  ├── name, code, type, level, province, city
  ├── is_985, is_211, is_double_first
  └── intro（院校介绍文本）
  用途: 院校列表、筛选、基本信息展示
  现状: 162 所（广东为主）

Layer 2: 录取数据 → AdmissionData 表 → ChromaDB
  ├── year, province, batch
  ├── major_name, min_score, min_rank
  ├── subject_requirements（选科要求）
  └── enrollment_quota（招生人数）
  用途: 推荐引擎的核心数据源——"你的分数能上什么专业"
  现状: 182K 文档，覆盖 3 省

Layer 3: 专业培养计划 → TenantData 表 → ChromaDB
  ├── major_name, college（所属学院）
  ├── duration（学制）, degree（学位）
  ├── core_courses（核心课程）
  └── objective（培养目标）
  用途: 回答"这个专业学什么"——学生最常问的问题
  现状: 几乎为空（需院校自行上传 Excel）

Layer 4: 就业数据 → TenantData 表 → ChromaDB
  ├── major_name, year
  ├── employment_rate（就业率）
  ├── avg_salary（平均薪资）
  ├── main_industries（主要行业）
  └── typical_companies（典型去向企业）
  用途: 回答"毕业能干什么、赚多少钱"——家长最关心的问题
  现状: 几乎为空

Layer 5: 行业知识库 → IndustryAnalysis 表 → ChromaDB global
  ├── industry_name, avg_annual_salary
  ├── employment_demand_index, entry_difficulty
  ├── pros, cons, outlook
  └── related_majors
  用途: 为推荐提供行业前景上下文
  现状: 10 个行业（curated，非实时）
```

### 4.2 数据扩展的具体目标

**从 3 省 → 10 省：**

当前覆盖：广东、北京、上海（~60 万考生覆盖）

需扩展：江苏、浙江、四川、湖北、山东、湖南、河南（+7 省）

扩展后覆盖：~800 万考生（60%+ 的全国考生）

**需收集的数据（按优先级）：**

| 优先级 | 数据层 | 具体内容 | 获取方式 |
|--------|--------|---------|---------|
| P0 | Layer 1 院校信息 | 10 省全部本科院校的基础信息 | 教育部公开名单 + 爬虫补充 |
| P0 | Layer 2 录取数据 | 10 省 × 近 3 年 × 所有本科专业的录取分数线+位次 | 各省教育考试院公开 API / 学信网 |
| P1 | Layer 3 培养计划 | 试点院校的详细培养方案 | 院校官网 + 院校上传 Excel |
| P1 | Layer 4 就业数据 | 试点院校的就业质量报告 | 院校官网 PDF + 院校上传 Excel |
| P2 | Layer 5 行业数据 | 更多行业 + 更新至 2025 年数据 | 国家统计局 + 招聘平台公开数据 |

### 4.3 试点院校（广东工业大学）需要收集的数据清单

以 gdufs 为例，确保单院校 demo 有完整数据：

| 数据类型 | 具体内容 | 当前状态 |
|---------|---------|---------|
| 院校基础信息 | 校名、代码、层次、省市、985/211/双一流、简介 | ✅ 已有 |
| 录取数据 | 2023-2025 年各专业在广东省的录取最低分+位次 | ⚠️ 部分有 |
| 录取数据 | 2023-2025 年各专业在其他省份的录取数据 | ❌ 缺失 |
| 培养计划 | 各专业核心课程、培养目标、学制学位 | ❌ 缺失 |
| 就业数据 | 各专业就业率、平均薪资、主要行业、典型企业 | ❌ 缺失 |
| 校园信息 | 宿舍条件、食堂、社团、奖学金、转专业政策 | ❌ 缺失 |

### 4.4 数据收集的技术路线

```
L0: 公开数据自动采集（已有基础设施）
  爬虫 (scrapers/) → 各省考试院 API → 公共数据池
  当前已有: eol_api.py (高考网), sunshine_gaokao.py (学信网)
  需扩展: 江苏/浙江/四川等省考试院 API 适配

L1: 院校自助上传（刚完成）
  Excel 模板 → 校验 → 入库 → ChromaDB 索引
  目标: 招生办 10 分钟内完成上传

L3: 数据库直连（未来高级版功能）
  对接正方/青果/强智等主流教务系统
  导出 SQL 模板 → 院校 IT 执行 → 导入
```
