# 招生报告页 Redesign — Design Spec

> 基于 brainstorming 确认。分支：`feat/admin-redesign-v2`

## 设计目标

去除现有页面的营销化呈现（大标题、口号、诗意文案），还原为真实的**数据推理工具页**。

核心功能：招生办老师查看"学生高频关注 → 现有内容缺口 → 改进建议"推理全链条，可编辑所有条目，可对比历史版本。

## 页面结构（自上而下）

### 1. 顶部工具栏

```
[报告周期 ▼] | 数据来源：近30天 AI 咨询记录          [编辑报告] [导出 PDF]
```

- 报告周期下拉：2026年5月 / 4月 / 3月
- 数据来源文字只读
- 编辑报告：切换三列进入可编辑模式（默认只读）
- 导出 PDF：导出当前报告

### 2. 策略视角 Tab

四选项纵排于三列上方：

```
策略视角
[本周总览] [内容优化] [宣讲重点] [渠道建议]
```

- 选中态：白底 + 蓝色文字 + 微阴影
- 未选中：透明背景 + 次要色文字
- 容器：浅灰背景 `#f1f5f9`，圆角 10px，内边距 3px

每个视角对应不同的三列数据（mock），切换时更新三列内容。

### 3. 三列数据卡片

每列结构相同：

```
┌──────────────────────────┐
│ 列标题 (14px 600)        │
│ 说明文字 (10px muted)   │          [+ 添加]
├──────────────────────────┤
│ ❶ 条目标题 (12px 600)   │
│   条目描述 (11px muted)  │
│ ─ ─ ─ ─ ─ ─ ─ ─ ─ ─ ─  │
│ ❷ ...                   │
│ ...                       │
│ ❹ ...                   │
└──────────────────────────┘
```

**三列内容**：

| 列 | 标题 | 序号色 | 说明 |
|---|---|---|---|
| 学生高频关注 | 按咨询频次排序 | 深蓝 #1e40af | 从咨询数据中提取的高频话题 |
| 现有内容缺口 | 按漏覆盖影响排序 | 琥珀 #faad14 | 知识库未覆盖或回答不充分的话题 |
| 改进建议 | 按优先级排列 · 可编辑修改 | 绿 #22c55e | 每个条目带 P0/P1/P2 优先级标签 |

**优先级标签颜色**：P0 红底 `#fee2e2` / P1 黄底 `#fef3c7` / P2 蓝底 `#dbeafe`

**编辑行为**：
- 默认三列只读
- 点击"编辑报告"后切换到可编辑模式
- 标题和描述使用 `contenteditable`，悬停显示虚线编辑边界
- "改进建议"列支持修改优先级标签
- 每列支持"+ 添加"新条目和删除（×按钮悬停出现）

**全部条目完整展示**：不设"升级解锁"折叠行，所有 4 条均可见（后续视角可能更多条目）。

### 4. 底部操作栏

```
上次保存：5分钟前 | 历史版本 (3)      [重置为系统建议]  [保存报告]
```

- **历史版本 (3)**：点击展开版本对比面板
- **重置为系统建议**：放弃修改，恢复到系统生成内容
- **保存报告**：保存当前编辑

## 历史版本对比面板

点击"历史版本 (3)"在页面底部展开：

```
┌─────────────────────────────────────────────────┐
│ ⏱ 报告历史版本  点击版本可查看该时间点的快照  [×] │
├────────────┬────────────────────────────────────┤
│ ▎当前修改   │ 对比：当前修改 ← → 系统建议         │
│  未保存     │                                     │
│             │ 改进建议 #1                          │
│  系统建议   │ ┌ 当前 ───────┐ ┌ 系统建议 ───────┐ │
│  05-21 09:30│ │发布《...推文》│ │发布《...推文》   │ │
│             │ └────────────┘ │ 用真实去向...     │ │
│  保存记录#2 │                └────────────────┘ │
│  05-20 16:15│                                     │
│             │ 改进建议 #2                          │
```

- 左侧版本列表：当前修改（蓝条高亮）、系统建议、历史保存记录

- 点击"还原到此版本"：将所有三列内容恢复为选中版本

## 数据（mock）

四个视角各自的 mock 数据结构：

```typescript
interface ReportEntry {
  rank: number
  title: string
  desc: string
}

interface ReportPerspective {
  concerns: ReportEntry[]
  gaps: ReportEntry[]
  actions: (ReportEntry & { priority: 'P0' | 'P1' | 'P2' })[]
}
```

四个视角：`weekly`（本周总览）/ `content`（内容优化）/ `lecture`（宣讲重点）/ `channel`（渠道建议，新增）

`channel` 视角数据新增 mock，聚焦渠道触达效果分析。

## 去掉的内容

- Hero 区（标题 + 副标题 + 口号文案）
- 推理链图示（学生问了什么 → 学校哪里没讲清楚 → 下一步该怎么做）
- "还有 N 条...（升级解锁）" 行
- 底部励志文案（"每一次咨询反馈..."）

## 实现要点

1. 修改 `ReportsPage.tsx`，保持在同一文件内
2. 新增 `channel` 视角的 mock 数据
3. 编辑状态用 `useState` 管理 `editing: boolean` + `editData` 副本
4. 历史版本用 `useState` 管理 `versions` 数组 + `selectedVersion` + `showHistory` boolean
5. 遵循 karpathy 原则：手术式改动，不引入新文件，不抽象历史版本为独立组件（保持内联，简单优先）
