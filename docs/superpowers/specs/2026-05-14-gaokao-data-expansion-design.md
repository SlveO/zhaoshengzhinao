# 高考志愿填报系统 — 数据扩展设计方案

## 一、目标

扩展知识库以支撑推荐系统多维度评分，收集广东省全部本科院校2020-2025年录取数据、完整专业目录、就业去向及行业分析数据。

**目标生源**：广东省学生
**目标院校**：广东省全部普通本科院校（~67所）
**数据年份**：2020-2025

## 二、数据维度

| 维度 | 数据项 | 推荐权重 |
|------|--------|---------|
| 院校信息 | 基本信息、排名、特色学科、校区 | 院校层次 10% |
| 专业目录 | 全部专业、学科门类、新增/停招标记 | 兴趣匹配 25% |
| 录取数据 | 2020-2025年广东省内分数线/位次/选科要求 | 分数匹配 30% |
| 就业去向 | 就业率、升学率、行业/岗位分布、薪资 | 就业前景 20% |
| 行业分析 | 景气度、薪资、趋势、从业者评价 | 就业前景+兴趣匹配 |
| 专业-行业映射 | 专业→行业→岗位链路 | RAG语义检索 |

## 三、采集架构

```
scrapers/
├── config.py              # 全局配置
├── base_scraper.py        # 基础爬虫类（限速/重试/日志）
├── sources/
│   ├── sunshine_gaokao.py # 阳光高考平台
│   ├── eol_api.py         # 掌上高考API
│   ├── gd_eea.py          # 广东教育考试院
│   ├── university_reports.py # 各校就业报告PDF
│   └── industry_data.py   # 行业数据
├── parsers/
│   ├── pdf_parser.py      # PDF解析
│   ├── html_parser.py     # HTML解析
│   └── data_cleaner.py    # 数据清洗
├── storage/
│   ├── schema.py          # 数据模型
│   ├── exporter.py        # JSON/CSV导出
│   └── validator.py       # 质量校验
├── run_all.py             # 并行采集主控
└── manual_import.py       # 手动导入工具
```

## 四、数据Schema

详见 `data/schema/` 目录下的JSON Schema定义。

- `colleges.json` — 院校信息
- `majors.json` — 专业目录
- `admissions.json` — 录取数据
- `employment.json` — 就业去向
- `industries.json` — 行业分析
- `major_industry_mapping.json` — 专业行业映射

## 五、数据源

| 数据类别 | 来源 | 方式 |
|---------|------|------|
| 院校+专业 | 阳光高考 gaokao.chsi.com.cn | HTTP+解析 |
| 录取分数 | 掌上高考 api.eol.cn + 广东教育考试院 | API+解析 |
| 就业去向 | 各校就业质量年度报告PDF | 下载+解析 |
| 行业薪资 | 国家统计局 + 招聘平台公开数据 | 结构化整理 |
| 从业者评价 | 知乎等公开讨论 | 文本采集 |

## 六、成功标准

- 覆盖广东省全部本科院校（≥67所）
- 每校所有招生专业全部收录
- 2020-2025年广东省内录取数据完整
- 新增/停招专业标记准确
- 就业数据覆盖主要专业
- 行业分析覆盖国民经济主要门类
- 数据可追溯至原始来源
